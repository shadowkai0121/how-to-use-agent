<interfaces>
This plan must lock these concrete authoring contract values:

```json
{
  "language": "zh-TW",
  "frontmatter": [
    {
      "id": "preface",
      "order": 0,
      "file": "book/frontmatter/00-前言.md",
      "title": "前言",
      "summary": "說明這本書的目標、讀者對象與學習方式。",
      "checkpoint": "你已理解本書的定位與閱讀方式。",
      "milestone": null,
      "authorNotes": ["保留為 README / EPUB 共用入口內容"],
      "outputs": ["readme", "epub"]
    }
  ],
  "chapters[*].file": "book/chapters/01-建立你的閱讀與第一個實作起點.md",
  "appendices[*].file": "book/appendices/附錄-a-術語表.md"
}
```

Current seeded entries are the preface, the known seven main chapters, and the glossary appendix. These are the required starting documents for Phase 2, not an upper bound. The schema and tests must allow future chapter additions, chapter splits, and reorderings beyond the current seven as long as:

```text
- manifest order values stay unique and strictly ascending within each section
- chapter filenames keep a sortable numeric prefix aligned to the current order
- the current known entries remain present after Phase 2
```

Allowed extended syntax is limited to:

```markdown
> [!NOTE]
> 說明文字

<details>
<summary>展開看完整說明</summary>

補充內容

</details>
```

Placeholder images must use this exact shape:

```markdown
![這裡預計放 GitHub Pages 部署完成畫面的截圖](../assets/placeholders/ch07-github-pages-live.png)
```
