<interfaces>
The tier contract is fixed as follows and must not be renamed:

```text
local   -> build:readme + build:epub + verify:drift + verify:links:local
pr      -> local + verify:placeholders + verify:links:pr + verify:epub
release -> clean dist/epub rebuild + pr tier + manual reader smoke reminder
```

Generated-output coverage is fixed as follows:

```text
scripts/verify-links.mjs
- local: README.md + book/frontmatter/**/*.md + book/chapters/**/*.md + book/appendices/**/*.md + root-relative/local asset targets
- pr: local coverage + regenerated README.md + dist/epub/cover.md referenced assets/links + full external URL scan
- release: pr coverage + fresh rebuild outputs, including dist/epub/how-to-use-agent.epub existence, dist/epub/how-to-use-agent.snapshot.json existence, and any generated Markdown/JSON asset references

scripts/verify-placeholders.mjs
- README.md
- book/frontmatter/**/*.md
- book/chapters/**/*.md
- book/appendices/**/*.md
- dist/epub/cover.md
- dist/epub/how-to-use-agent.snapshot.json
```

Failure summaries must use these exact headings when applicable:

```text
LINK VALIDATION FAILED
PLACEHOLDER GATE FAILED
EPUB VALIDATION FAILED
PHASE 3 GATE FAILED
```

The PR workflow must run on `pull_request` and invoke:

```bash
pnpm install --frozen-lockfile
node scripts/verify-phase3.mjs --tier pr
```

The release workflow must run on both `workflow_dispatch` and `release.published`, invoke:

```bash
node scripts/verify-phase3.mjs --tier release
```

and upload:

```text
dist/epub/how-to-use-agent.epub
dist/epub/how-to-use-agent.snapshot.json
```
</interfaces>
