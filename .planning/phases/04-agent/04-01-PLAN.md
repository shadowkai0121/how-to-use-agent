<interfaces>
Phase 4 must standardize on these exact script names in `package.json`:

```json
{
  "test:phase-04": "node --test tests/phase-04/*.test.mjs",
  "verify:phase4": "node scripts/verify-phase4.mjs",
  "check:phase-04": "npm run validate:authoring && npm run verify:phase4 && npm run test:phase-04 && npm run build:readme && npm run build:epub && node scripts/verify-drift.mjs"
}
```

Every Phase 4 chapter contract must enforce these exact headings in reading order:

```markdown
## 本章產出
## 你會練到什麼
## Prompt
## Command
## Checkpoint
## 卡關先這樣回正
```

The checkpoint contract is fixed as:

```text
- first bullet: observable result or page/output the reader should see
- second bullet: repo evidence or command that proves current state
- final line: whether it is safe to continue or when to stop
```

The recovery contract is fixed as:

```text
- must directly tell Agent to return to the previous checkpoint
- must name the file(s) that should be kept
- must ask for the remaining minimum steps instead of a full rewrite
```
